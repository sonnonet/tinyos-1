Interface: tos.chips.atm128rfa1.BusPowerManager

interface BusPowerManager

Commands
command void configure(uint16_t startup, uint16_t keepalive) Sets the startup and keepalive timeout values in milliseconds.
command void releasePower() Releases the power up request.
command void requestPower() Requests to power up this bus.

Events
event void powerOff() This event is called when there are no outstanding power requests and the keepalive timeout value has expired.
event void powerOn() This event is called when the bus is powered on and the maximum specified startup time has elapsed.

Commands - Details

configure

command void configure(uint16_t startup, uint16_t keepalive)

Sets the startup and keepalive timeout values in milliseconds. Only the largest numbers are remembered, as the bus might not be operational if one of the chips on it is not operational.

releasePower

command void releasePower()

Releases the power up request. You must call this command exactly the same number of times as you have called requestPower. If other chips still want to keep the power (e.g. a measurement is in progress), then powerOff will not be called.

requestPower

command void requestPower()

Requests to power up this bus. The bus might be already powered up, in which case the powerOn event will not come.

Events - Details

powerOff

event void powerOff()

This event is called when there are no outstanding power requests and the keepalive timeout value has expired.

powerOn

event void powerOn()

This event is called when the bus is powered on and the maximum specified startup time has elapsed. Implementations should initialize the chip or go into power down mode.
